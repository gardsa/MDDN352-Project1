Your Weather, Your Way

Aims

To provide a weather forecast experience that is personal, local, and easy. 4C will offer users the ability to customise their own interface, thereby stripping all the irrelevant numbers and graphs that don't make sense and displaying what YOU want to see.

Target Audience

Everyone. Much of the weather information 4C will provide is already accessible across many apps and websites all over the world. The point of difference is that the service will appeal to everyone, as they have control over what they do and don't see.

Technical Construction

4C will be built as a web application, meaning users will not have to download an app, so that it can be used on as many devices as possible. Therefore, the languages and frameworks used to build it will be HTML5, CSS3 and Javascript. It will harness relevant Javascript libraries for the modular user interface functionality, as well as the APIs of Metservice and BloomSky to retrieve the weather data.

Visual Direction

While users will be able to select from a number of different themes, the default colour scheme is comprised of two contrasting tones. The green and purple colours will form the main UI, while a light grey will add some contrast for primary elements. White has been selected for the large, informative textual elements and the purple will be used for more secondary text.

Two sans-serif fonts will be used. This choice was fairly evident, as serif fonts do not fit into the simple, minimal and informative aesthetic 4C aims to achieve. Noto Sans will serve as the main font-family, used for paragraphs and smaller text elements. Roboto is a very strong bold font, and will be used for larger text elements, such as the temperature or rainfall weather components.

Clickable Mockup

Interact 4C aims to be simple, user-friendly, and highly customisable. This mockup prototype is intended to give you an idea of what the interface might be like. It is interactive, and certain parts will allow you to navigate 4C in its initial form. The menu buttons, weather components, and settings tab are all clickable.

Each of the weather components will display the current weather information for that particular part of the forecast. Tapping on them will then display a long-range forecast - around 6-12 hours ahead. The main menu allows the user to add, remove, and navigate between saved locations. Clicking on the options icon in the top right lets the user select their relevant weather components, as well as reshuffle the layout of their interface.

Modular Interface Functionality

Modular design requires breaking down a design into small parts (modules), designing each independently, and then later combining them into a larger system. Modular UI and UX patterns will be followed to create 4C. This means creating a structured UI system, and understanding how its components can be used to create the user flow. This will allow users to customise the interface to fit their needs.

Interact Drag and drop the weather component modules to experience how 4C will utilise the Packery and Draggabilly Javascript libraries to achieve a modular UI.

Users will also be able to resize the modules, and select their primary weather component.
